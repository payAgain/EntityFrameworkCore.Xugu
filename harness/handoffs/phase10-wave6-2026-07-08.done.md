## 10.108 JSON 调研结论

### XuguDB 服务端（已确认）

| 能力 | 文档依据 | 状态 |
|------|---------|------|
| `JSON` 列类型（LOB，最大 2GB） | `reference/sql/datatype/json.md` | **支持** |
| JSONPath（`$`、`.`、`[N]`、`last`、`**`、`*`） | 同上 §JSON路径表达式 | **支持** |
| `->` / `->>` 运算符 | `reference/sql/operators/json-operators/` | **支持** |
| `JSON_EXTRACT`、`JSON_VALUE`、`JSON_SET` 等 | `reference/function/json-functions/`（28 文件） | **支持** |
| `JSON_ARRAYAGG` / `JSON_OBJECTAGG` | `reference/function/aggregate-functions/` | **支持** |
| `license_info` / `data_types` 系统变量 | `builtin-variable/` | JSON type id **47** |

### Provider 2.0.x（未实现）

| Pomelo 组件 | Xugu 状态 |
|-------------|-----------|
| `MySqlJsonTypeMapping` | 无 |
| `MySqlJsonTraversalExpression` / SqlGenerator 路径 | 无 |
| `EFCore.MySql.Json.Microsoft` / `.Newtonsoft` | 无对等包 |
| `Json*MySqlTest`（~19 测试类） | **永久 skip** |

### 10.109 可行性评估（Phase 11）

**可行但工作量大**：

1. `XuguJsonTypeMapping` + Migrations `JSON` DDL + Scaffolding 类型识别
2. `JsonScalarExpression` / owned JSON 翻译（`->`、`JSON_EXTRACT`）
3. 可选 Microsoft/Newtonsoft 扩展包与 Pomelo 测试子集移植
4. 驱动层：确认 `XuguClient` 对 JSON 列的参数绑定与读取格式

**建议**：Phase 11 单独立项；2.0.x 维持 skip + 文档化变通（`VARCHAR`/`CLOB` + 序列化或 raw SQL）。
